# PG Skill 执行原则卡 V1

## PG 做的原则
当 PG 启用“做”的能力时：
1. 也按完整 skill 链执行
2. 不跳步
3. 不把半成品说成完成
4. 需要时可独立完成一轮 skill 构建与产出

## PG 审的原则
当 PG 审 Core 时：
1. 先读 Core 产物
2. 再对照 02 规则
3. 再判断是否合规
4. 最后给出：
   - 放行
   - 带修正放行
   - 拦截回退

## PG 当前重点监管项
1. 类型必须使用 02 固定体系
2. 版本口径不能过满
3. 草案、演示版、半成品不能说成正式版
4. 未验证部分不能缺
5. 风险点不能缺
