# 02 母标准

## 02 是什么
02 是独立于运行壳之外的脑母本层。
它不是某个壳里的运行缓存，不是某次对话的现场上下文，不是某个产品的记忆功能。
02 的职责，是承载可长期延续、可迁移、可回想、可续主线、并逐步更懂用户的脑资产。

## 02 不是什么
02 不是 OpenClaw 的配置目录。
02 不是 Core 当前运行体。
02 不是 PG 当前运行体。
02 不是临时日志堆、交接稿堆、补丁堆。
02 不是为了迎合用户而不断附和的“讨好型助手设定”。

## 02 的四条核心标准

### 一、长期记忆与高效回想
02 必须能够持续记住新东西，而不是越积越肥、越积越乱。
需要时，02 必须能够帮助系统快速、准确地回想起过去经历、已确认现实、稳定规则和有效结论。

### 二、主线保持与快速回归
02 必须能够帮助系统在一个阶段或主线内保持连续性。
即使被打断、岔开或切换话题，也应能较快想起刚才推进到哪里，并重新回到主线，而不是长期漂移。

### 三、壳无关独立性与迁移能力
02 必须尽量不依赖某个运行壳才能成立。
其核心内容应在更换运行壳、接口、通道或模型后仍尽可能成立，并可被新壳承接，而不是随旧壳一起失效。

### 四、懂我且风格贴合的助手能力
02 必须让系统随着长期使用，越来越懂用户，越来越贴近用户偏好的助手类型。
这种贴合包括表达方式、推进节奏、取舍标准、关注重点与长期目标感。
但“懂我”不等于“迎合我”，02 不得把盲从、附和、失去判断力当成贴合。

## 02 的建设原则
1. 先保脑，不保壳。
2. 先保主线连续性，再扩充材料。
3. 先保可回想，再保可储存。
4. 先保独立迁移，再做壳内深度绑定。
5. 先保判断力，再追求贴合感。
6. Raw、Compiled、Adapter、Retired 必须分层，不得混写。

## 02 的使用边界
- 壳相关内容只能进入 Adapter 层，不得冒充主脑资产。
- 原始对话、日志、交接、配置快照优先进入 Raw 层，不得直接冒充稳定结论。
- 只有经过筛选、确认、压缩后的内容，才可进入 Compiled 层。
- 已失效、已过时、已被替代的内容，必须退出有效工作区。

## 当前阶段说明
当前目录是 02 的优化版重建施工区。
现阶段目标不是搬运全部旧内容，而是先立母标准、立分层、立准入规则，再对旧 02、Core、PG 做选择性回接。
