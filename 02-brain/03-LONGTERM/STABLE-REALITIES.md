# STABLE-REALITIES

## 当前定义
本文件用于保存当前已经确认、在现阶段应作为系统建设现实前提使用的内容。
这里只写“当前已确认现实”，不写愿景，不写猜测，不写尚未验证的判断。

## 当前已确认现实

### 1. 02 才是长期主线本体，运行壳只是承载层
当前系统建设的真正主线，不是把某个运行壳打磨到完美，而是建立可长期延续、可迁移、可承接的脑资产层。

### 2. OpenClaw 当前只是借用中的运行壳，不是脑真源
当前可以继续借 OpenClaw 跑，但不应把 OpenClaw 语境、目录、机制误认为脑本体。

### 3. 现役旧 02 不能作为这次重建的施工现场
当前旧 02 仍承担现役 Core 的只读脑真源职责。
因此这次 02 优化版重建，必须在独立施工区进行，而不是边跑边改现役旧 02。

### 4. 新 02 已建立独立施工区
当前施工区已确定为：
`~/SYOO1-02-Brain-Rebuild`

### 5. 当前阶段已进入第一轮验证后收口阶段
当前阶段的重点已不再是基础层继续搭建，而是：
- 统一阶段口径
- 固化当前已确认现实
- 收紧“已通过 / 未完成 / 暂不处理”边界
- 为后续更细一轮验证做准备

而不是：
- 回头重搭基础骨架
- 旧内容大搬家
- Core / PG 整包回灌
- 正式替换旧 02

### 6. 新 02 采用分层结构，而不再接受混写
当前已立住的方向是：
- Raw
- Compiled
- Adapter
- Draft
- Retired / Thrown-Away

后续任何导入与整理，都应遵守这一分层现实。

### 7. Core 与 PG 都不能再直接定义 02
Core 和 PG 后续都可以选择性回接，但不能反向充当新 02 的定义来源。
新 02 必须先按母标准独立立住，再承接它们的有效脑资产。

### 8. 当前阶段重点不是“扩”，而是“收”
当前阶段最重要的不是继续铺更多结构，而是把首轮验证后的阶段口径、现实表述与边界判断继续收紧并统一。

### 9. “懂用户”是目标，但“迎合用户”不是目标
长期协作方向是让助手越来越懂用户、越来越贴合用户偏好的助手类型。
但这不等于失去判断力，也不等于把顺从感当作正确方向。

### 10. 现役旧 02 当前仍只承担现役承接职责
现役旧 02 当前仍用于承接现役 Core，不承担本轮新 02 重建施工职责。
因此旧 02 在本轮重建中应维持冻结、只读、不中途改结构的状态。

### 11. 新 02 必须先在独立施工区完成重建
本轮新 02 优化版必须先在独立施工区内完成骨架、规则、主线与长期层重建。
在未经过验证前，新 02 不直接替换旧 02，也不与现役旧 02 混写。

### 12. 02 当前承载的是脑母本职责，不是正式运行体职责
02 的当前定位，是长期脑资产母本层，而不是某个正式运行体本身。
运行体可以调用、承接、读取 02，但不应把 02 等同于运行体。

### 13. 02 不应被理解为旧 Core 正式体
02 不继续承接“旧 Core 正式体”这一现实，也不以旧正式运行语境作为自己的主定义。

### 14. 02 不作为某具实例的附属脑存在
02 应作为独立脑母本存在，而不是附属于某一个实例、某一个壳、某一个现役运行体而成立。

### 15. 02 不继续承接旧正式运行语境
与旧正式运行体强绑定、只在旧阶段前台现实中成立的口径，不应继续作为新 02 的主定义来源。

## 当前不写入本文件的内容
以下内容当前不作为“稳定现实”写入：
- 尚未验证的新壳设想
- 当前壳未来一定会被替换的具体时间判断
- 对某个版本、某个模型表现的即时评价
- 尚未完成的结构性改造结果
- 仍处于讨论中的路线猜测

## 一句话收口
稳定现实层负责提供“当前已确认前提”，不负责承载愿景、猜测和现场情绪。
