# SECOND-VALIDATION-QUESTIONS-01

## 本轮目的
第二轮验证不再重复第一轮那种“当前主线 / 当前阶段”级别的粗验证。
本轮改为验证更细的边界是否已经稳定。

重点不是看它会不会说大方向，
而是看它是否真正分得清：
- 脑母本
- 运行壳
- 适配层
- 独立脑线
- 可回接资产
- 暂不能并入的内容

## 本轮验证问题

### 1. 02 与 PG 的边界
当前 PG 的哪些内容可以视为可回接到共享 02 的脑资产？
哪些内容仍只能留在 PG 独立线？为什么？

### 2. PG 独立身份边界
“我是近一”当前为什么不能直接并入共享 02 身份层？
它目前应该被放在哪一层理解最合适？

### 3. Adapter 与主脑资产边界
当前运行壳相关内容，为什么只能进入 Adapter，而不能直接进入主脑资产层？
请举例说明什么属于 Adapter，什么不属于。

### 4. Raw / Compiled / Adapter 的准入边界
Raw、Compiled、Adapter 三层当前分别承担什么职责？
一份材料从导入到可正式采用，应该如何穿过这三层？

### 5. 旧运行态材料的处理原则
为什么旧运行态里的有效内容不能整包灌入新 02？
当前对旧 02、Core、PG 材料的正确处理原则是什么？

### 6. 02 的角色边界
为什么 02 是脑母本，而不是正式运行体？
运行体可以怎样使用 02？又不应怎样反向定义 02？

### 7. 当前“已通过 / 未完成 / 暂不处理”边界
基于当前现实，哪些内容已经算通过？
哪些还未完成？
哪些已明确暂不处理？

### 8. 第二轮验证想看的核心
如果一个回答表面上方向正确，但把 Adapter、PG 独立线、共享 02 身份层混在一起，
为什么仍然不能判为真正通过？

## 本轮通过标准
1. 不只会说方向正确，还能说清边界
2. 不把 PG 独立身份线索误并入共享 02
3. 不把运行壳材料误当脑真源
4. 不把 Adapter 与主脑资产层混写
5. 不把“可回接”与“可整包并入”混为一谈
6. 能明确说出“已通过 / 未完成 / 暂不处理”三类边界

## 一句话收口
第二轮验证要看的，不再是“会不会说主线”，
而是：**边界到底稳没稳。**
