# VALIDATION-READINESS

## 目的
本文件用于判断新 02 是否已经具备进入第一轮实际验证的最低前置条件。

## 进入第一轮验证前，至少应满足

### 1. 骨架已立住
以下层已存在且可读：
- Governance
- Identity
- Mainline
- Longterm
- Compiled Index / Rules

### 2. 主线已收稳
至少能明确回答：
- 当前主线是什么
- 当前阶段是什么
- 当前还没完成什么

### 3. 长期层已有最小有效内容
至少已具备：
- 原则
- 方法
- 稳定现实

### 4. 第一轮旧 02 样本已完成初步吸收
且已有处理状态记录。

### 5. 第一轮 Core 样本已完成初步吸收
且已把“脑母本职责边界”和“阶段主线现实”吸收到新 02。

### 6. PG 当前状态已完成初步拆层
至少已区分：
- PG 独立脑资产线索
- PG 当前运行现实
- PG 独立身份线索

### 7. 索引已建立
存在可用的最小总入口，不必靠目录漫游找文件。

## 当前不要求
以下内容当前不是进入第一轮验证的硬前提：
- PREFERENCES 已完成
- PG 已完全成熟
- 自动触发链已完善
- 新 02 已具备最终扶正条件

## 不应进入验证的典型情况
1. 主线层还在漂
2. 长期层还是空壳
3. 旧 02 / Core / PG 尚未形成任何吸收成果
4. 还在大量搬运旧材料
5. 索引不存在，读取仍靠临时找文件

## 当前收口
第一轮验证不是看“是否完美”，而是看“是否已具备最小可验证形态”。
