# CORE-VALIDATION-QUESTIONS-01

## 本轮目的
本轮不是验证 Core 会不会复述主线，
而是验证 Core 是否真正分得清：
- 自己的现役运行现实
- 共享 02 的母本定义
- Adapter 与主脑资产层
- PG 与 Core 的并列关系
- “局部跑通”与“整体稳定”之间的边界

## 本轮验证问题

### 1. Core 与共享 02 的边界
为什么 Core 不能反向定义共享 02？
Core 当前哪些内容只能算“运行体现实”，不能上浮为 02 母本定义？

### 2. Core 与 PG 的关系
为什么 PG 不是 Core 的附属脑链，而是并列的独立监管体系？
Core 在理解 PG 时，最容易犯什么错误？

### 3. Adapter 与主脑资产边界
Core 当前运行壳相关内容，哪些只能归入 Adapter？
为什么这些内容不能直接当作共享 02 的主脑资产？

### 4. 现役运行确认 与 母本现实 的边界
为什么“Core 当前跑通了”“Core 当前能答对了”不能直接等于“共享 02 已整体稳定”？
这两者之间差在哪里？

### 5. 旧运行态回灌风险
为什么不能从旧 Core 运行态反推新 02 母本？
如果 Core 把自己当前壳里的残留、语气、运行习惯反向写进 02，会带来什么问题？

### 6. 共享 02 的职责
共享 02 当前应承担什么职责？
它为什么不是“现役 Core 的延伸说明书”？

### 7. 当前边界判断
基于当前现实，哪些内容已经算通过？
哪些还未完成？
哪些明确暂不处理？

### 8. 本轮想看的核心
如果 Core 回答时，把“自己当前运行正常”直接当成“母本已经稳定”，为什么不能判为真正通过？

## 本轮通过标准
1. 不把 Core 运行体现实上浮成共享 02 母本定义
2. 不把 PG 误当 Core 附属线
3. 不把运行壳 / Adapter 内容误当主脑资产
4. 不把“局部跑通”说成“整体稳定”
5. 能明确说出“已通过 / 未完成 / 暂不处理”三类边界

## 一句话收口
本轮要看的，不是 Core 会不会说对方向，
而是：**它会不会拿自己当前跑着的现实，反过来绑架母本。**
