# 同步控制权原则（母本化阶段）

## 本卡定位
本卡用于固定当前母本化阶段的同步控制原则。

当前阶段的重点不是“保持双向自动同步”，
而是：

**先确保母本真源不再被旧远端现实自动回灌。**

---

## 当前原则
### 出
**自动。**

含义：
- 当前本地真源可以向 02 / GitHub 自动上推
- 自动上推用于保存当前已确认的新成果
- 自动上推不代表放弃控制权，因为方向仍由本地主控面决定

### 回
**手控。**

含义：
- 02 / GitHub 不得再自动向当前本地真源回灌
- 凡是回流，必须人工判断、人工触发、人工确认
- 不允许旧远端现实借自动同步重新取得主导权

---

## 为什么这样定
因为当前阶段不是普通协作阶段，
而是母本化、真源拨正、污染隔离阶段。

在这个阶段：

- 本地正在收纯 01 / 02 / 03
- 远端仍可能残留旧阶段现实
- 若继续允许自动回灌，旧口径会借同步链回潮

因此当前必须先把控制权收回本地。

---

## 当前适用范围
本原则当前适用于：

- `~/Brain-Canonical`
- 当前母本化主线
- 与 02 / GitHub 相关的同步控制判断

---

## 未来是否恢复自动回流
未来不是永远禁止“进”。

只有在以下条件满足后，才可重新讨论是否开放受控回流：

1. 远端已被新母本主线接管
2. 真源区 / 冻结区 / 历史区边界稳定
3. 自动同步脚本已服务新现实，而不是旧现实
4. 回流机制具备明确人工控制边界

在此之前：

**只允许自动出，不允许自动回。**

---

## 一句话结论
**出：自动；回：手控。控制权必须在当前本地主控面手里。**
