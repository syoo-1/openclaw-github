# 主脑唯一与可回滚原则

## 第一原则：一个主脑
- 系统始终只允许一个主脑真源。
- 当前唯一主脑真源为：`~/Brain-Canonical`
- 任何模型、任何壳、任何入口，都只是接入层，不是主脑本体。
- 不允许制造第二真源，不允许让 session / sqlite / workspace 取代主脑。

## 第二原则：系统必须可回滚
- 任何系统改造，都必须建立在可回退、可恢复、可接管的前提上。
- 任何新壳、新模型、新接入方式，在未验证稳定前，不得替代现有可用入口。
- 备份、恢复、封存、回退，不是附属能力，而是底线能力。

## 当前判断
- 主脑唯一性：长期原则，不因模型或壳变化而变化。
- 可回滚能力：长期原则，不因当前阶段策略而变化。

## 一句话版本
主脑必须唯一，系统必须可回滚。
