# 最小读取与节约 Token 原则

## 本卡定位
本卡用于固定当前主线中的读取原则，避免大脑越用越臃肿、入口越读越长、上下文越烧越快。

---

## 核心目标
在保持连续性、判断力、长期记忆能力的同时：

- 优先读最少必要内容
- 减少无效展开
- 防止长历史压过当前主线
- 节约 token
- 提高接班与推进效率

---

## 默认最小读取
继续当前主线时，默认优先读取：

1. `~/Brain-Canonical/START-HERE-MOTHERBASE.md`
2. `~/Brain-Canonical/CURRENT.md`
3. `~/Brain-Canonical/01-02-03-04-总纲.md`
4. `~/Brain-Canonical/02-脑母本定义.md`
5. `~/Brain-Canonical/04-成品实例定义.md`

---

## 按需展开
只有在当前问题明确需要时，才继续展开：

- `longterm/`
- `handoff/`
- `recovery/`
- `identity/`
- 冻结区 / 历史区

---

## 默认不优先读取
以下内容默认不应优先进入当前上下文主平面：

- 冻结区
- 历史交接长文
- 旧 Core 时代文件
- 与当前主线无关的旧运行事故材料
- 只解释历史、不能推动当前主线的长文

---

## 使用原则
1. 先短入口，再长材料
2. 先当前态，再历史态
3. 先真源，再冻结区
4. 先推动主线，再补充背景
5. 只在必要时展开长内容，以节约 token

---

## 一句话结论
**先读最小必要入口，按需展开长材料；保连续，不臃肿，省 token。**
