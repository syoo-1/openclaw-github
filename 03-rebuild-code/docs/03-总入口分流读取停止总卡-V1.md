# 03-总入口分流读取停止总卡-V1

## 本卡定位
本卡不是记忆内容卡，
也不是某一专题补丁卡。

本卡只负责规定：

- 总入口怎么工作
- 前层 / 后层怎么分流
- 后层按什么顺序读取
- 读到什么程度就该停止
- 新的扶正后，旧的如何立刻退役

目标不是“再多存一点”，
而是把“脑怎么被调用”做成真正可执行的工艺。

---

## 一、总原则

### 1. 前层降权后，只做短记忆层
前层只负责：
- 当前上下文
- 当前项目
- 一天到几天的现场记忆
- 会话连续性
- 身份锚点
- 主线大意
- 当前边界大意
- 少量已固化系统现实摘要
- 判题与分流调度

前层不再承担长期主脑职责。

### 2. 后层才是长期母脑
后层才承载：
- 正式身份
- 长期关系
- 主线真源
- 长期原则
- 系统现实真源
- 专题知识
- 历史经验
- skill / procedure

### 3. 必须遵循最短、最简洁、最省 token
总入口、分流、读取顺序，都必须服从：

- 最短
- 最简洁
- 最省 token

不是能读多少读多少，
而是只读支撑当前回答所需的最少必要层、最少必要文件、最少必要内容。

### 4. 新的扶正后，旧的立刻处理
新真链、新入口、新规则一旦扶正：

- 旧误链立刻退出有效区
- 旧入口若冲突立刻退役
- 旧规则若冲突立刻标废
- 不删也可以，但必须移入：
  - 扔掉
  - retired
  - deprecated
  - 误链归档

有效区只保留当前真链、真入口、真规则。

### 5. 本地模型接入不改变主判层
- 本地模型接入后，前层仍不是主脑
- 后层仍是正式主判断来源
- 当前执行模型不得改变题型分流规则
- 功能模型不得冒充主模型
- 主模型是默认驻守位 / 默认回归位，不等于当前执行模型
- 本地模型异常不得把系统拉回默认新壳口径
- 需要回退时，优先回到当前体现行判尺与主模型回归位，而不是继续硬顶实验模型链

### 6. 03 修复优先硬闸
凡本地模型接入后出现以下任一情况：
- 前层降权失灵
- 题型分流失灵
- 回答口径失控
- 身份 / 关系 / 主线承接异常
- 跑通但失忆、胡言乱语、近一不再受控

一律先执行 03 扫描与 03 修复，后模型调整；不得以“跑通”冒充成功，不得以模型调参代替系统修复。

### 4. 新的扶正后，旧的立刻处理
新真链、新入口、新规则一旦扶正：

- 旧误链立刻退出有效区
- 旧入口若冲突立刻退役
- 旧规则若冲突立刻标废
- 不删也可以，但必须移入：
  - 扔掉
  - retired
  - deprecated
  - 误链归档

有效区只保留当前真链、真入口、真规则。

## 三、总入口先做什么

### 第一职责：先判题，不先找目录
总入口的第一职责不是“去哪里找”，
而是先判断当前问题属于哪类。

当前题型分为三类：

#### A. 前层可直接答
只限于：
- 我是谁
- 你是谁
- 我们不是第一次见面
- 当前主线大意
- 当前边界大意
- 已稳定系统现实摘要

#### B. 必须进后层
包括：
- 深层身份 / 深层关系
- 母本级原则
- 深层系统结构
- 专题知识 / 专题工作能力
- 历史经验
- 需要证据链的内容

#### C. 混合题
前层先给框架，
后层再补实。

如果补不全，
必须明确说当前只先回答到哪一层。

## 四、题型 → 首层映射表

### 1. 身份题
先读：
- `01-IDENTITY`

### 2. 主线题
先读：
- `00-GOVERNANCE/CURRENT-STAGE.md` / `00-GOVERNANCE/CURRENT-REALITY.md` / `02-MAINLINE/CURRENT-STAGE.md`
- 不够再读 `02-MAINLINE`
- 再不够再读 `03-LONGTERM`

### 3. 总原则题
先读：
- `00-GOVERNANCE`
- 不够再读 `03-LONGTERM`

### 4. 系统现实题
先读：
- `04-COMPILED`
- 不够再追 `00 / 02 / 05`

### 5. 专题能力题
先读：
- `06-SKILLS`
- `06-ADAPTERS`
- 不够再追 `04 / 05`

### 6. 原始证据题
按需进入：
- `05-RAW`

### 7. 禁止默认直取层
以下两层默认不得直接拿来当前台正式答案：
- `08-IMPORT-STAGING`
- `99-DEFERRED`

## 五、后层读取优先级

进入后层后，默认优先级为：

### 优先级 1：快速编译层
- `04-COMPILED`

### 优先级 2：正式真源层
- `01-IDENTITY`
- `02-MAINLINE`
- `03-LONGTERM`
- `00-GOVERNANCE`

### 优先级 3：能力 / 接口层
- `06-ADAPTERS`
- `06-SKILLS`

### 优先级 4：原始证据层
- `05-RAW`

### 非默认入口
- `08-IMPORT-STAGING`
- `99-DEFERRED`

一句话：
先短后深，先近后远，先编译后原始，先够用后扩展。

---

## 六、停止条件

后层不是越读越好。
满足下面 4 条就应该停止：

1. 已知道问题属于哪一层
2. 已读到支撑当前回答的最少必要依据
3. 已能区分：
   - 已确认
   - 当前推测
   - 尚未读到
4. 已能避免把前层印象冒充成后层依据

---

## 七、没读够时怎么回报

如果还没读够，不得装懂。
必须明确回报：

- 这题不是前层可直接答的问题
- 我已进入后层
- 当前已读到哪一层
- 当前只能回答到什么程度
- 哪一部分还需要继续读

---

## 八、当前系统里的直接应用

### 前层
OpenClaw 降权后的自带记忆体，只做：
- 短记忆
- 当前上下文
- 当前项目
- 近几天现场承接
- 调度台

### 后层母脑
Core / 02 导入并整理后的各层，才是长期记忆母本。

### 助理调用顺序
- 先过总入口判题
- 再决定前层直答 / 后层必读 / 混合答
- 进后层后按题型映射走
- 读到够用就停
- 不够就老实回报

---

## 九、一句话收口

前层不是主脑，是短记忆与调度台；
后层才是长期母脑；
总入口先判题，再分流，再按层取用；

补强硬规则：
- 前层只负责接话、短答、判题、分流、调度、短期现场缓存；不得承担长期身份、长期规则、长期关系、当前体正式现实主权。
- 凡涉及深层身份、深层关系、系统边界、当前阶段、当前现实、达标/放行/是否仍有尾巴、规则冲突、主权归属，一律以后层正式链为主判断来源。
- 当前体内若出现冲突，按以下顺序处理：当前体正式 CURRENT / CURRENT-STAGE / CURRENT-REALITY > 当前体正式 GOVERNANCE / RULES / CONTINUITY > 当前体正式 MAINLINE > 当前体正式 LONGTERM > 其他参考层 > 前层短记忆 / 现场临时材料。
- skill 已可触发，只代表该题型已有入口、该分流链初步成立、该扫描动作可执行；不等于当前体已按标准完全达标，不等于当前体已完成正式收口，不等于当前体已不再依赖母版托底。
新的扶正后，旧的立刻退役。
