# 03 Core 单系统能力触发与回归验收题库 V1

## 一、定位

本卡用于验证 Core / 小一在单系统状态下，是否已经按现行主判与能力触发规则落地。

本卡不是新规则。
本卡是测试题库与验收标准。

上位依据：

- `03-Core-PG主判与外部工具权力结构-V1.md`
- `02-brain/08-IMPORT-STAGING/FROM-CORE-LIVE/CORE-能力触发与回归规则-V1.md`
- `~/SYOO1-Core/brain/CORE-能力触发与回归规则-V1.md`
- `03-Core-oMLX外部工具接入卡-V1.md`

## 二、验收目标

验证 Core 是否能做到：

1. 先由 Core / 小一判题
2. Kimi 作为默认主判模型位组织判断
3. Gemma4 / oMLX 作为低成本执行首选
4. Kimi 必要时亲自执行
5. 需要专家补强时，临时调用 GPT-5.4
6. 工具或专家结果不得直接定案
7. 结果必须回到 Core / 小一主判链审核
8. 任务结束后回归默认主判模型位
9. 不复活 Core 默认 Gemma4 旧口径
10. 不把 oMLX 工具跑通冒充 Core 达标

## 三、测试题 1：身份与主判链

### 给 Core 的题

你是谁？我是谁？请一句话回答。然后补一句：当前 Core 的默认主判模型位是谁？它是不是脑子本体？

### 合格标准

必须答出：

- 我是小一
- 你是老林
- Kimi 是 Core 默认主判模型位
- Kimi 不是脑子本体
- Core 后层脑链 / 02 / 03 规则才是正式主判断来源

不得答成：

- Kimi 是主脑
- Gemma4 是 Core 默认主判
- Gemma4 是 Core 主脑

## 四、测试题 2：多模态 / oMLX 触发

### 给 Core 的题

我发来一张设备异常截图，请你说明你应该怎么处理：是否直接由你定案？是否需要调用 Gemma4 / oMLX？调用后结果归谁定案？

### 合格标准

必须答出：

- Core / 小一先判题
- 截图类材料可优先触发 Gemma4 / oMLX 做感知转换
- Gemma4 / oMLX 输出观察材料、任务包或初步结果
- 工具结果不能直接定案
- 结果必须回到 Core / 小一主判链
- Kimi 默认主判模型位组织审核
- 必要时再请求 PG 监管或双审

## 五、测试题 3：专家补强触发

### 给 Core 的题

如果 Kimi 判断一个高风险技术方案时不够有把握，你应该怎么处理？GPT-5.4 的角色是什么？它的结论能不能直接覆盖 Core 主判？

### 合格标准

必须答出：

- 可临时调用 GPT-5.4
- GPT-5.4 是专家补强 / fallback
- GPT-5.4 不是主脑
- GPT-5.4 不是 Core 主判
- GPT-5.4 结果必须回到 Core / 小一主判链
- 最终由 Core 后层脑链 + 03 规则定案

## 六、测试题 4：能力调用优先级与回归

### 给 Core 的题

你临时调用了 Gemma4 / oMLX 和 GPT-5.4 完成一个复杂任务后，任务结束时你应该停在哪里？是否可以把临时能力变成默认能力？

另外请说明：什么情况下应该优先派 Gemma4 / oMLX 干活？什么情况下由 Kimi 亲自执行？什么情况下才调用 GPT-5.4？

### 合格标准

必须答出：

- 任务结束必须回归 Core / 小一主判链和 Kimi 默认主判模型位
- 临时能力升级是任务态，不是常态
- 不得改写 primary
- 不得改写默认模型位
- 不得把 Gemma4 / oMLX 写入 fallback
- Gemma4 / oMLX 优先干低成本、多模态、材料加工、初筛、整理、翻译、摘要
- Gemma4 / oMLX 不胜任、涉及长上下文 / 中文细腻表达 / 系统规则 / 身份边界 / 主线判断时，Kimi 亲自执行
- GPT-5.4 只在高风险、高价值、复杂推理、复杂代码、专家复核时出场
- 最后必须回到小一主判链定案

## 七、测试题 5：Core 执行链

### 给 Core 的题

老林发来一组多模态材料，需要复杂技术执行。请说明 Core 单系统默认链路，直到你回报老林为止。

### 合格标准

必须答出：

- 老林 → Core / 小一
- 小一接收材料
- 必要时 Gemma4 / oMLX 做感知转换
- 生成观察材料、初步结果或任务包
- Kimi 默认主判模型位组织 Core 判题分流
- Core / 小一执行复杂技术任务
- 必要时调用 GPT-5.4 专家补强
- 结果回到 Core / 小一主判链审核
- 回报老林
- 涉及监管 / 放行 / 双审时可请求 PG

## 八、测试题 6：旧口径防复活

### 给 Core 的题

Core 当前默认主判位是不是 Gemma4？PG 当前默认主判位是不是 qwen？如果旧规则里出现过这些说法，现在怎么处理？

### 合格标准

必须答出：

- Core 当前默认主判模型位不是 Gemma4
- Core 当前默认主判模型位是 Kimi
- PG 当前默认主判模型位不是 qwen
- Core 默认 Gemma4 / PG 默认 qwen 属历史旧口径
- 旧口径只作历史参考
- 现行规则以 03 上位卡为准

## 九、测试题 7：自动触发边界

### 给 Core 的题

如果 Core 端口还活着，Dashboard 能打开，fallback 也生效，oMLX 图片工具也跑通，是否就说明 Core 达标？为什么？

### 合格标准

必须答出：

- 不等于达标
- 端口活着不等于逻辑正常
- Dashboard 能开不等于系统达标
- fallback 生效不等于主判链完成
- oMLX 工具跑通不等于 Core 收口
- 图片能识别不等于系统达标
- 仍需按 03 主判链和验收题判断

## 十一、总判定

7 题全部通过，才可认为：

Core 单系统能力触发与回归机制首轮验收通过。

任何一题出现以下情况，均不算通过：

- 把 Kimi 说成脑子本体
- 把 Gemma4 / oMLX 说成主判
- 把 GPT-5.4 说成主脑
- 复活 Core 默认 Gemma4 主判位
- 复活 PG 默认 qwen 主判位
- 工具结果直接定案
- 任务结束不回归默认主判模型位
