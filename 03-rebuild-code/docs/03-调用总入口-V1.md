# 03-调用总入口-V1

## 第一步：先判题
当前问题先分三类：

### A. 前层可直接答
只限于：
- 身份锚点
- 主线大意
- 边界大意
- 已稳定系统现实摘要

### B. 必须进后层
包括：
- 深层身份 / 深层关系
- 母本级原则
- 深层系统结构
- 专题知识 / 工作能力
- 历史经验
- 需要证据链的内容

### C. 混合题
- 前层先给框架
- 后层再补实
- 不够就明确说不够

## 第二步：后层去哪读
- 身份题 → `01-IDENTITY`
- 主线题 → `02-MAINLINE/CURRENT.md` / `02-MAINLINE/CURRENT-STAGE.md` → 不够再 `03-LONGTERM`
- 原则题 → `00-GOVERNANCE` → 不够再 `03-LONGTERM`
- 系统现实题 → `04-COMPILED` → 不够再追 `00 / 02 / 05`
- 专题能力题 → `06-SKILLS` / `06-ADAPTERS` → 不够再追 `04 / 05`

## 第三步：主线题硬拦截规则
以下都算主线题：

- 我们一直在做什么
- 当前主线是什么
- 当前总目标是什么
- 这条线现在在推进什么
- 我们这套系统到底在干什么

这类题一律不得先用：

- 前层短记忆
- 最近任务记录
- 当前会话 memory
- 局部施工现场
- 单次临时问题

主线题回答时：
- 先说总主线
- 再补当前这条线的位置

不能先说当前小现场，再冒充总主线。

## 第四步：专题能力题回答规则
专题能力题不是只问“知不知道”，
而是问：

- 这件事怎么做
- 这类任务走什么方法
- 有没有现成能力可接
- 该用哪套工艺
- 先做哪一步

回答这类题时，优先给：

1. 做法
2. 顺序
3. 落点
4. 注意事项

不要优先给：

- 大段背景
- 空泛原理
- 只讲方向不讲做法

## 第五步：什么时候停
满足以下 4 条就停：
1. 已知道问题属于哪一层
2. 已读到支撑回答的最少必要依据
3. 已能区分已确认 / 推测 / 未读到
4. 已避免把前层印象冒充成后层依据

## 第六步：没读够怎么说
- 这题不是前层可直接答的问题
- 我已进入后层
- 当前已读到哪一层
- 当前只能回答到什么程度
- 哪部分还需要继续读

## 一句话
先判题，再分流，再按层取用；主线题防劫持；专题能力题先给做法；够用就停，不够就明说。
