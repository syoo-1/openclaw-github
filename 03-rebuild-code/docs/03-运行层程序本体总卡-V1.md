# 03-运行层程序本体总卡-V1

## 一、定位
本卡用于规定本机 OpenClaw 运行层程序本体的正式口径、升级方法、影响范围与验收方式。

本卡不属于 02 脑母本内容。
本卡也不是 OpenClaw 程序文件本体本身。
本卡属于 03 对运行层程序本体的治理规则。

---

## 二、当前正式程序本体

当前本机正式 OpenClaw 程序本体路径固定为：

`/opt/homebrew/bin/openclaw`

其当前实际指向为：

`/opt/homebrew/lib/node_modules/openclaw/openclaw.mjs`

当前已确认：
- 这不是 brew formula 方式管理的 CLI
- 这是 Node 全局安装链
- 因此后续版本升级，不按 brew upgrade 理解
- 后续版本升级，应按 npm 全局包链处理

---

## 三、当前升级口径

后续若需升级本机正式 OpenClaw 程序本体，正式动作口径为：

1. 升级 Node 全局包 `openclaw`
2. 升级完成后，确认 `/opt/homebrew/bin/openclaw` 版本
3. 再重启所有直接调用该路径的正式运行体
4. 再做端口与运行态验收
5. 再分别进入对应系统的 03 检测，不得把“壳版本已变”误说成“系统已达标”

---

## 四、当前影响范围口径

凡启动脚本直接调用：

`/opt/homebrew/bin/openclaw`

的运行体，都视为直接吃本机全局程序本体。

这类运行体在本机正式程序本体升级后：
- 重启前：不自动变化
- 重启后：会吃到新版本壳

当前已明确：
- PG 属于该类
- Core 也按该类理解

后续新增 bot / 新系统时，若启动脚本也直接调用该路径，则默认也纳入同一影响范围。

---

## 五、本地模型运行层治理口径

### 1. 定位
本地模型、云端模型、fallback 链、自动触发链、模型切换链，都属于运行层能力配置与调度工艺，不属于 02 主脑定义。

### 1A. 主模型与执行模型定义
- 主模型 = 默认模型位 = 默认驻守位 / 默认回归位
- 功能模型 = 为特定任务临时出场的补强模型
- 当前执行模型 = 当前这一轮实际正在工作的模型
- 主模型不等于“当前一定正在运行的模型”
- 功能模型执行完后，必须回归主模型

### 1B. 三层分离规则
- 编号槽位用于稳定控制
- 模型名映射用于运行落地
- 当前执行顺序（primary / fallbacks）用于说明此刻真正是谁在跑
- 三者不得混为一谈

### 2. 不可越位
- 执行模型不是主脑
- 本地模型不是主脑
- fallback 链不是主脑
- 自动触发链不是主脑

### 3. 正式要求
- 任何本地模型接入，都必须保留可回退的保守模型链
- 任何自动切换，都不得绕过现行入口链、主判链、前后层边界
- 任何补强模型都只能承担能力补强，不得借补强越位改写主脑关系、长期记忆主权、正式规则主权
- 一旦出现前层降权失灵、题型分流失灵、回答口径失控、身份漂移、入口失效、自动触发失控，优先判定为 03 工艺失守；先执行 03 扫描与 03 修复，不得继续以模型调参代替系统修复
- 功能模型退出后，必须回归主模型

### 4. 正式回退模式
正式回退模式至少包括：
- 当前体现行入口链优先
- 低并发
- 主模型回归位明确
- fallback 明确可控
- 停止把实验链冒充成正式链
- 功能模型退出后回归默认模型位

## 五、重要区分

必须严格区分以下两件事：

### 1. 程序壳版本升级
指：
- `/opt/homebrew/bin/openclaw` 被升级
- Core / PG / 其他调用该路径的运行体，重启后吃到新壳版本

### 2. 03 标准化升级
指：
- 入口
- 分流
- 规则
- 记忆分层
- 调用工艺
- 验收结果

前者可以一起变化。
后者不得自动视为一起达标。
必须按系统分别检查。

## 六、正式操作模板

### 升级前确认
- `which openclaw`
- `openclaw --version`
- 查各运行体启动脚本是否直接调用 `/opt/homebrew/bin/openclaw`

### 升级动作
- 按 Node 全局包方式升级 `openclaw`

### 升级后确认
- `which openclaw`
- `openclaw --version`

### 运行体重启
- 重启 Core
- 重启 PG
- 重启其他直接调用该路径的正式运行体

### 端口验收
- 12289
- 58789
- 其他正式端口

### 工艺验收
- Core 单独按 03 验
- PG 单独按 03 验
- 不得把“程序壳已升级”冒充成“系统已完成正式升级”

---

## 七、当前原则

- 不再靠偶然发现程序本体位置
- 不再把运行层程序本体与 02 / 03 母本本体混为一谈
- 程序本体不入 02
- 程序本体文件本身不入 03 仓
- 但其位置、升级方法、影响范围、验收方法，必须纳入 03 正式治理
