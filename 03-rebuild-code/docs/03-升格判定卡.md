# 03-升格判定卡

## 本卡的定位
本卡用于规定：  
从壳内前层经历池导入后的材料，在经过缓冲、分流、整理后，什么情况下允许升格，升到哪里，依据什么判断。

本卡不负责抽取。  
本卡也不负责定义 02 的主权本体内容。  
本卡只负责：升格判定。

---

## 一、总原则

### 1. 升格不是“感觉差不多”
材料不能因为：

- 看起来重要
- 看起来像原则
- 看起来很顺手
- 当前很热
- 当前很常用

就直接升格。

### 2. 升格必须晚于缓冲与分流
壳内材料必须先经过：

- `08-IMPORT-STAGING`
- 分类判断
- 分流整理
- 必要提炼

之后才谈升格。

### 3. 升格的本质是“主权确认”
一旦升格进入：

- `01-IDENTITY`
- `03-LONGTERM`
- `00-GOVERNANCE`

就不再只是阶段性材料，  
而是开始进入 02 的正式主权结构。

### 4. 宁可慢一点，不要错升
错升的代价，大于慢升。  
因为一旦把临时东西升成长期真相，后面会污染整个主脑。

---

## 二、可升格目标层

### 1. `01-IDENTITY`
适合承接：

- 身份定义
- 用户关系底座
- 稳定称谓
- 长期稳定互动口径
- 明确长期成立的人物关系现实

### 2. `03-LONGTERM`
适合承接：

- 长期原则
- 长期偏好
- 稳定现实
- 长期有效的结构性结论
- 跨阶段仍成立的经验结晶

### 3. `00-GOVERNANCE`
适合承接：

- 准入规则
- 治理边界
- 分层口径
- 升格规则
- 工程硬约束
- 母本施工规则

### 4. 留在 `04-COMPILED`
适合承接：

- 已有提炼价值
- 已经不是原始素材
- 但还不适合升为长期正式层
- 适合作为中间成果、候选结果、复用块

---

## 三、升格判定五问

任一材料拟升格前，至少问这五个问题：

### 1. 它是否跨阶段仍成立？
如果离开当前阶段、当前任务、当前热度后就可能失效，  
一般不能直接升长期层。

### 2. 它是否已经脱离“临时现场口径”？
如果它仍明显带有：

- 当下应急口径
- 当前壳侧口径
- 临时实验口径
- 临时项目语境

一般不能直接升格。

### 3. 它是否已经足够稳定？
如果这条内容：

- 还可能很快反转
- 还缺证据
- 还在观察期
- 还没有经过至少一轮验证

一般不宜直接升格。

### 4. 它升格后是否会影响长期判断？
如果升格后会改变：

- 身份理解
- 用户关系理解
- 长期原则
- 治理边界
- 母本施工方式

就必须更谨慎，不能草率。

### 5. 它更像长期真相，还是更像中间成果？
如果它更像：

- 当前阶段总结
- 当前问题答案
- 某次试错结论
- 某轮过渡方案

那通常更适合先留在 `04-COMPILED` 或 `99-DEFERRED`。

---

## 四、各目标层的具体升格标准

### A. 升入 `01-IDENTITY` 的标准

只有当材料同时接近以下特点时，才考虑升入 `01-IDENTITY`：

1. 与身份或关系直接相关
2. 不是临时称呼、临时场景口径
3. 已在多轮对话或多阶段中保持稳定
4. 升格后有助于长期认人、认关系、认互动边界
5. 不依赖某个特定壳或某个临时项目

典型可升内容：

- 稳定身份定义
- 稳定用户称谓
- 长期成立的关系口径
- 长期稳定互动底座

---

### B. 升入 `03-LONGTERM` 的标准

只有当材料同时接近以下特点时，才考虑升入 `03-LONGTERM`：

1. 已超出当前阶段
2. 具有跨任务复用价值
3. 不只是某次现场经验，而是可复用结论
4. 在未来较长时间内大概率仍成立
5. 对主线连续性和长期理解有价值

典型可升内容：

- 长期偏好
- 长期原则
- 稳定现实
- 多轮试错后沉下来的结构性经验

---

### C. 升入 `00-GOVERNANCE` 的标准

只有当材料同时接近以下特点时，才考虑升入 `00-GOVERNANCE`：

1. 它描述的是“怎么管”“怎么分”“怎么准入”“怎么施工”
2. 它不是一次性技巧，而是可复用治理规则
3. 它的适用范围明显高于某个单次项目
4. 它对母本建设与长期秩序有直接影响
5. 不写进去就容易反复踩坑或长期混线

典型可升内容：

- 准入规则
- 分层规则
- 施工冻结规则
- 升格规则
- 工程硬约束

---

## 五、不宜直接升格的内容

以下内容通常不宜直接升格：

- 某次壳侧临时总结
- 某轮调试中的临时口径
- 当前很热但尚未稳定的信息
- 某次试错刚得出的结论
- 仍强依赖当前版本壳结构的内容
- 只在某次现场成立的判断
- 过于具体、过于短期、过于应急的方案

这些内容更适合先进入：

- `04-COMPILED`
或
- `99-DEFERRED`

---

## 六、推荐判定路径

### 路径 1：原始材料
`08-IMPORT-STAGING`
→ 分类
→ `05-RAW` / `02-MAINLINE` / `04-COMPILED` / `99-DEFERRED`

### 路径 2：候选升格材料
`04-COMPILED`
→ 五问判定
→ 决定是否升入 `01-IDENTITY` / `03-LONGTERM` / `00-GOVERNANCE`

### 路径 3：暂缓材料
`99-DEFERRED`
→ 后续补证
→ 再判断是否转入 `04-COMPILED` 或升格

---

## 七、禁止事项

禁止：

- 跳过 `08-IMPORT-STAGING` 直接升格
- 因为“现在很好用”就升格
- 因为“看起来像长期原则”就升格
- 把壳侧临时口径直接升为主脑长期规则
- 把未验证内容升入 `01-IDENTITY`
- 把短期调试经验直接升入 `03-LONGTERM`
- 把一次性操作技巧直接升入 `00-GOVERNANCE`

---

## 八、验收标准

本卡成立后，至少应满足：

1. 知道升格不是主观感觉，而有判断门槛
2. 知道什么更适合进 `01-IDENTITY`
3. 知道什么更适合进 `03-LONGTERM`
4. 知道什么更适合进 `00-GOVERNANCE`
5. 知道什么应该先留在 `04-COMPILED` 或 `99-DEFERRED`
6. 后续整理时，不会把“当前热信息”误升成长期真相

---

## 九、一句话定义

升格不是把重要内容往上堆；  
而是把经过缓冲、分流、提炼、验证后，  
真正具备长期成立条件的材料，  
按类型正式送入 02 的主权层。
