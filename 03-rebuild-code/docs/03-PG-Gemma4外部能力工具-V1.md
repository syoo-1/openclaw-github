# 03-PG Gemma4 外部能力工具 V1

## 一、定位

Gemma4 是 PG 的外部能力工具。

Gemma4 不进入 PG 模型槽。
Gemma4 不是近一。
Gemma4 不是主脑。
Gemma4 不是裁判。
Gemma4 不拥有主判断权。

## 二、当前状态

当前已确认：

- Gemma4 本体可通过 llama.cpp 运行
- 模型文件为：
  ~/LLM-Shell/models/gemma4-e4b-it-q4_k_m.gguf
- 调用脚本为：
  ~/SYOO1-SCRIPTS/pg-call-gemma4.sh
- 03 固化副本为：
  ~/System-Snapshots/03-rebuild-code/tools/pg-call-gemma4.sh
- 手动调用链已跑通
- 单轮调用后可自动退出
- thinking 输出已被压住
- 近一可先判定、再调用、再审核、再回报

## 三、适用任务

Gemma4 可处理：

- 文本整理
- 摘要
- 改写
- 结构化
- 要点提取
- 大白话转换
- 非主权类辅助分析

## 四、禁止任务

以下任务不得交给 Gemma4 裁定：

- 身份判断
- 关系判断
- 主权归属
- 当前阶段
- 当前现实
- 达标判断
- 放行结论
- 系统边界裁定
- PG / Core / 02 / 03 主脑关系裁定

遇到这些任务，必须回到：

- 近一 / Kimi
- PG 正式主脑链
- 002-PG-03-达标扫描
- 03 正式工艺卡

## 五、调用流程

标准流程：

1. 近一先判定任务是否适合交给 Gemma4
2. 适合时，用 exec 调用：
   ~/SYOO1-SCRIPTS/pg-call-gemma4.sh
3. Gemma4 只完成具体任务
4. 近一审核 Gemma4 输出
5. 近一用自己的话回报老林

Gemma4 输出不得直接作为最终裁定。

## 六、当前边界

当前状态为：

手动调用链已跑通。

尚未完成：

- 自动触发
- 自动分流
- 自动路由
- 长期稳定性观察

因此不得说 Gemma4 已正式接入 PG 自动能力链。

## 七、硬结论

Gemma4 作为 PG 外部能力工具，手动调用链已跑通。

它只处理文本整理、改写、摘要、结构化等非主权任务。

达标、放行、身份、关系、主线、阶段、系统边界问题不得交给 Gemma4 裁定，必须回到近一 / Kimi / 002。
