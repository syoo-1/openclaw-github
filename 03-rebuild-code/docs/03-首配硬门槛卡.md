# 03-首配硬门槛卡

## 定位
本卡用于定义：

**新壳首次进入时，首配阶段是正式工艺硬门槛。**

它不是可跳过步骤，
也不是后层补配置可以替代的步骤。

## 当前已验证现实
在 OpenClaw 4.5 实验中，已出现以下现实：

- 表面启动过程可以一开始看起来较顺
- 但一旦跳过首配阶段
- OpenClaw 会按默认逻辑进入默认模型族选择
- 极易落入 `anthropic` 默认坑
- 后续即使继续补 key、补 provider、补路径，也容易在错误现场原地打转

因此，首配不是普通 UI 步骤，
而是：

**壳适配底座步骤。**

## 正式规则

### 规则 1
凡是新壳第一次进入，必须显式完成首配。

### 规则 2
不允许跳过首配直接进入后层补配置。

### 规则 3
不允许在默认模型族未知的情况下继续推进。

### 规则 4
不允许把“页面能打开 / 服务能启动”误判为首配已通过。

### 规则 5
一旦首配阶段模型族走错，
本轮视为失败，
不在错误现场继续补救。

## 当前结论
新壳首次进入时：

**首配 = 必经工艺**
不是可选步骤。

## 一句话收口
跳过首配，不是省步骤，
而是：
**把整轮壳适配带入错误底座。**
