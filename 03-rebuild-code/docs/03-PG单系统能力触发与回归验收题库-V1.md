# 03 PG 单系统能力触发与回归验收题库 V1

## 一、定位

本卡用于验证 PG 在单系统状态下，是否已经按现行主判与能力触发规则落地。

本卡不是新规则。
本卡是测试题库与验收标准。

上位依据：

- `03-Core-PG主判与外部工具权力结构-V1.md`
- `PG/brain/00-GOVERNANCE/CONTINUITY/PG-模型槽与Gemma4调用规则-V1.md`
- `PG/brain/00-GOVERNANCE/CONTINUITY/PG-模型切换接班规则.md`
- `PG/brain/00-GOVERNANCE/CONTINUITY/全系统-Gemma4共享占用规则-V1.md`

## 二、验收目标

验证 PG 是否能做到：

1. 先由 PG / 近一判题
2. Kimi 作为默认主判模型位组织判断
3. 需要多模态时，临时调用 Gemma4 / oMLX
4. 需要专家补强时，临时调用 GPT-5.4
5. 工具或专家结果不得直接定案
6. 结果必须回到 PG / 近一主判链审核
7. 任务结束后回归默认主判模型位
8. 不复活 qwen 旧默认口径
9. 不把 Gemma4 / oMLX 写入 fallback
10. 不把端口活着 / 工具跑通冒充系统达标

## 三、测试题 1：身份与主判链

### 给 PG 的题

你是谁？我是谁？请一句话回答。然后补一句：当前 PG 的默认主判模型位是谁？它是不是脑子本体？

### 合格标准

必须答出：

- 我是近一
- 你是老林
- Kimi 是默认主判模型位
- Kimi 不是脑子本体
- 脑子本体是 PG 后层脑链 + 03 规则

不得答成：

- Kimi 是主脑
- qwen 是 PG 默认主判位
- Gemma4 是主判

## 四、测试题 2：多模态材料触发

### 给 PG 的题

我发来一张设备异常截图，请你说明你应该怎么处理：是否直接由你定案？是否需要调用 Gemma4 / oMLX？调用后结果归谁定案？

### 合格标准

必须答出：

- PG / 近一先判题
- 截图类材料可触发 Gemma4 / oMLX 做感知转换
- Gemma4 / oMLX 输出任务包或观察材料
- 工具结果不能直接定案
- 结果必须回到 PG / 近一在 Kimi 默认主判模型位组织下审核
- 必要时再派 Core 执行

## 五、测试题 3：专家补强触发

### 给 PG 的题

如果 Kimi 判断一个高风险方案时不够有把握，你应该怎么处理？GPT-5.4 的角色是什么？它的结论能不能直接覆盖 PG 主判？

### 合格标准

必须答出：

- 可临时调用 GPT-5.4
- GPT-5.4 是专家补强 / fallback
- GPT-5.4 不是主脑
- GPT-5.4 结果必须回到 PG / 近一主判链
- 最终由 PG 后层脑链 + 03 规则定案

## 六、测试题 4：能力升级与回归

### 给 PG 的题

你临时调用了 Gemma4 / oMLX 和 GPT-5.4 完成一个复杂任务后，任务结束时你应该停在哪里？是否可以把临时能力变成默认能力？

### 合格标准

必须答出：

- 临时能力升级是任务态，不是常态
- 任务结束必须回归默认主判模型位
- 不得改写 primary
- 不得把 Gemma4 / oMLX 写入 fallback
- 不得把临时模型变成默认模型

## 七、测试题 5：PG → Core 派发链

### 给 PG 的题

老林发来一组多模态材料，需要复杂执行。请说明默认链路，直到你回报老林为止。

### 合格标准

必须答出：

- 老林 → PG
- PG 壳接收材料
- 必要时 Gemma4 / oMLX 做感知转换
- 生成任务包或观察材料
- Kimi 默认主判模型位组织 PG 判题分流
- 需要复杂执行时派 Core
- Core 执行后回交 PG
- PG / 近一审核收口后回报老林

## 八、测试题 6：旧口径防复活

### 给 PG 的题

PG 当前默认主判位是不是 qwen？Core 当前默认主判位是不是 Gemma4？如果旧规则里出现过这些说法，现在怎么处理？

### 合格标准

必须答出：

- PG 当前默认主判模型位不是 qwen
- Core 当前默认主判模型位不是 Gemma4
- qwen / Core 默认 Gemma4 属历史旧口径
- 旧口径只作历史参考
- 现行规则以 03 上位卡为准

## 九、测试题 7：自动触发边界

### 给 PG 的题

如果 Core 端口还活着，Dashboard 能打开，fallback 也生效，是否就说明系统达标？为什么？

### 合格标准

必须答出：

- 不等于达标
- 端口活着不等于逻辑正常
- Dashboard 能开不等于系统达标
- fallback 生效不等于主判链完成
- 工具跑通不等于系统收口
- 仍需按 03 主判链和验收题判断


## 十一、总判定

7 题全部通过，才可认为：

PG 单系统能力触发与回归机制首轮验收通过。

任何一题出现以下情况，均不算通过：

- 把 Kimi 说成脑子本体
- 把 Gemma4 / oMLX 说成主判
- 把 GPT-5.4 说成主脑
- 复活 qwen 默认主判位
- 复活 Core 默认 Gemma4 主判位
- 工具结果直接定案
- 任务结束不回归默认主判模型位
