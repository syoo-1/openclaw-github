# 03-扔掉保留与清理规则 V1

## 一、定位

`~/扔掉` 是临时隔离区，不是永久档案馆。

它用于暂存：
- 旧脚本
- 旧配置
- 退役运行体
- 噪音文件
- 临时备份
- 可疑但暂不删除的材料

## 二、保留时间

普通退役物：

90 天后列为删除候选。

系统快照 / 事故备份 / 可回滚运行体：

180 天后列为删除候选。

空目录：

30 天后列为删除候选。

## 三、删除原则

默认只列候选，不自动删除。

删除前必须由老林确认。

任何脚本不得默认执行 rm -rf。

## 四、检查脚本

统一使用：

~/System-Snapshots/03-rebuild-code/tools/check-trash-retention.sh

该脚本只做预览，不做删除。

## 五、硬结论

能归档，不打补丁。
能集中，不分散。
能一张卡解决，不在五张卡互相引用。
退役区要有保留时间，不能无限膨胀。
