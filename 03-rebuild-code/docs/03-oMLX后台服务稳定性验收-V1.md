# 03 oMLX 后台服务稳定性验收 V1

## 目标
验证 oMLX 作为本地多模态服务层时，是否具备基础后台运行稳定性。

## 测试环境
- 服务：oMLX
- 地址：http://127.0.0.1:18000
- 模型：gemma4-e4b-4bit
- 启动方式：后台方式启动，不使用 brew services
- 模型目录：~/MLX/oMLX/models
- 模型实体：~/MLX/Gemma4-e4b/hf-cache

## 本次已确认
1. 端口仍在监听：127.0.0.1:18000
2. 进程仍在运行
3. 已运行时间超过 30 分钟
4. /v1/models 正常返回 gemma4-e4b-4bit
5. 文本接口 3 连测通过
6. 图片接口 3 连测通过
7. 服务未崩溃、未退出

## 日志确认
日志显示模型已加载：

Loaded model: gemma4-e4b-4bit

模型估算大小：

5.10GB

## 速度参考
图片测试中，300 tokens 输出约 10.55～12.21 秒。
约 24.6～28.4 tok/s。

## 已知提示
日志出现：

Structured output requires xgrammar.

当前判定：
- 不影响普通文本调用
- 不影响图片说明
- 不影响当前多模态材料包方向
- 暂不安装 --with-grammar，避免增加 torch / xgrammar 复杂依赖

## 当前结论
oMLX 后台服务稳定性首轮验收通过。

## 验收口径
可视为：
- oMLX 后台服务可运行
- 模型可被发现并加载
- 文本和图片接口可连续调用
- 当前可进入 PG/Core 外部工具接入前的封装阶段

## 接入前硬边界
1. 不使用 brew services 常驻启动，先保持手动后台启动
2. 不直接写入 PG/Core 主配置
3. PG/Core 只能通过包装脚本调用
4. oMLX 挂掉时，PG/Core 不得被拖死
5. 失败时必须返回“工具失败”，而不是污染主判断链
