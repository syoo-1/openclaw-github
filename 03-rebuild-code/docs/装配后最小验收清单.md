# 装配后最小验收清单

## 目标
用于验证由 01 + 02 + 03 生成出的新系统，
是否已经达到最小可应用状态，
并且没有把运行态残留误抬为母本定义。

## 最小必验项
1. 已生成实例自己的 `.env.local`
2. 已生成实例自己的 `runtime/start-*.sh`
3. 已生成 `POST-ASSEMBLY-SECRETS.md`
4. `runtime/.openclaw/openclaw.json` 中端口已改为实例端口
5. `runtime/.openclaw/openclaw.json` 中 workspace 已指向实例自身
6. `runtime/workspace/BOOTSTRAP.md` 已指向该实例应承接的正式脑来源
7. 启动脚本可执行
8. 未把真实密钥写入 01 / 02 / 03 母本
9. 未把运行态残留反写污染 01 / 02 / 03 母本
10. 未把旧测试口、旧施工区、旧运行时口径残留带入新实例

11. 新壳首次适配时，首配已显式完成，未跳过首配
12. 若目标为双模型 / 多模型，已在首配阶段一次性正确完成
13. gateway token 已在正确时序下获取、注入并验证，不只是页面可开

## 通过标准
- 结构正确
- 脑挂接正确
- 启动补件齐全
- 密钥口径正确
- 母本未受运行态污染
- 可继续进入启动验收 / 剥离独立阶段

## 一句话收口
新实例是否合格，不只看能不能启动，
还要看：
**有没有把运行态残留错带进母本和正式体。**
