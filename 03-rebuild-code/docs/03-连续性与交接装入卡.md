# 03-连续性与交接装入卡

## 目的
新系统拼装时，除了装壳、装脑、装首配，还必须装入连续性与交接骨架。
否则系统虽能启动，但主线续接、模型切换接班、短接班与连续性验收会缺位。

## 必装位置
当前正式规则来源为：

`~/System-Snapshots/02-brain/00-GOVERNANCE/CONTINUITY/`

至少应包含：
- 交接原则.md
- 模型切换接班规则.md
- 最小交接格式.md
- 连续性验收标准.md

## 拼装要求
1. 新系统拼装后，必须带入以上连续性/交接规则骨架
2. 不允许只装壳与脑，而缺少连续性规则
3. 不允许把旧 Core 的 handoff / trigger_handoff / status 旧材料直接当正式规则装入
4. 旧材料只可作为提炼参考，不得代行正式主权
5. 当前正式交接主权属于 02 正式内围规则，不属于旧运行遗产

## 最小验收
拼装完成后，至少确认：
1. 系统能说明当前主线程
2. 系统能说明当前停点与下一步
3. 模型切换时，能先交接、再确认、再继续
4. 未接住时，能明确说未稳定接住
5. 轻度换题后，仍能拉回主线

## 禁止
- 禁止把旧阶段现场状态板当正式规则装入
- 禁止把 trigger 临时层当长期正式层
- 禁止把旧 carrier 相关残留当正式连续性规则
