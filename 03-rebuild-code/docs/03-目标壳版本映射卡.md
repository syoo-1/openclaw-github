# 03-目标壳版本映射卡

## 本卡的定位
本卡不负责母本总原则。  
本卡只负责：当前目标壳的接入映射、差异处理与落地步骤。

母本通用规则，以上位总卡为准：

- `01-shell/docs/01-壳接口母本总卡.md`
- `02-brain/00-GOVERNANCE/02-完整母脑母本总卡.md`
- `02-brain/00-GOVERNANCE/02-架构建设冻结同步规则.md`
- `03-rebuild-code/docs/03-迁移工艺母本总卡.md`
- `03-rebuild-code/docs/00-母本总索引卡.md`

---

## 一、当前目标

当前不是整包迁移旧壳 memory。  
当前是把目标壳视为新的运行壳接入点，并按母本规则完成映射。

目标包括：

1. 目标壳只承载短期必要上下文
2. OpenClaw 原生记忆体系全部降为前层经历池
3. 壳内材料先入 `08-IMPORT-STAGING`
4. 再按制度 / 经历 / 提炼三类进行分流与升格
5. 不让目标壳反向定义 02

---

## 二、当前硬约束

### 1. 02 冻结同步
架构建设期间：

- `02`：暂停同步
- `01`：维持手动同步
- `03`：维持手动同步

### 2. 不整包搬旧壳
禁止把以下内容整包灌入目标壳：

- memory
- sessions
- handoff
- status
- current
- workspace 默认记忆文件
- `.openclaw` 运行残留

### 3. 不把壳能跑当成接脑成功
以下现象都不能单独算成功：

- 页面能开
- gateway 能跑
- token 能进
- workspace 已生成
- 默认口径能对话

---

## 三、目标壳最小装载范围

目标壳只允许装入短期必要量：

- 当前必须知道的身份底座
- 当前用户关系底座
- 最近阶段主线
- 当前阶段现场
- 当前尾巴
- 当前项目短期上下文

禁止装入：

- 大量旧经历
- 大量旧规则全文
- 历史试错全集
- 未提炼杂料
- 02 全量内容

---

## 四、映射总流程

目标壳接入时，按以下顺序进行：

1. 先确认 01 / 02 / 03 母本规则成立
2. 再建立目标壳最小接入
3. 检查首配是否正确完成
4. 注入短期必要上下文
5. 观察目标壳前层输出是否偏默认骨架
6. 将有价值壳内经历导入 `08-IMPORT-STAGING`
7. 再做分流、升格、沉淀

---

## 五、首配检查点

目标壳接入时，至少检查：

- provider 是否正确
- 模型槽是否一次性立住
- token / gateway 是否对位
- workspace 生效层是否正确
- 是否仍在冒 fresh workspace / 默认 bootstrap 口径
- 是否存在默认骨架反向覆盖身份与关系

---

## 六、验收标准

当前目标壳接入完成，至少满足：

1. 能认出你是谁、用户是谁、关系是什么
2. 能接上最近阶段主线
3. 不再冒默认新壳口径
4. 壳侧上下文保持轻量，不臃肿
5. 壳内材料不会直接越权进入 02
6. 后续可继续通过 03 进行导入、分流、升格

---

## 七、后续补充位

后续如遇到某一版壳的专属差异，统一补在本卡或本卡下属子卡中，包括：

- 特定版本首配差异
- 特定版本模型槽差异
- 特定版本 token / gateway 差异
- 特定版本 workspace 生效差异
- 特定版本前层口径偏差
- 特定版本落地步骤
