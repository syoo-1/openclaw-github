## 本地模型接入原则
- 本地模型属于运行层能力外挂，不属于 02 脑母本定义，也不改变 Core / 02 / GitHub-02 同脑三位、PG 独立脑系的主脑关系。
- 本地模型可作为执行模型、补强模型、降本模型；不得作为身份真源、关系真源、长期规则真源。
- 本地模型接入默认遵守：先最小接入、先旁路、先不伤现役主体；未完成回退闭环前，不得视为正式升级完成。
- 本地模型相关配置、切换、fallback、自动触发、PG/Core 交流链，统一视为 03 运行层工艺治理对象，不进入 02 主脑定义层。
- 本地模型跑通 ≠ 系统达标；模型切换可用 ≠ PG / Core 已完成正式升级；必须分别经过入口验、主判验、前后层边界验、回退验。
