## 二、三母本分工

### 01：壳接口母本
01 负责：

- 壳接入规则
- 首配规则
- 模型槽规则
- token / gateway / workspace 规则
- 壳侧前层记忆边界
- 测试壳与正式壳隔离

一句话：
**01 负责让壳正确接入并服从 02。**

### 02：完整母脑母本
02 负责：

- 制度内容
- 经历内容
- 提炼内容
- 长期主权
- 身份与关系底座
- 长期原则与稳定现实

一句话：
**02 是长期主权真源。**

### 03：迁移工艺母本
03 负责：

- 抽取
- 分流
- 升格
- 映射
- 跨版本迁移
- 目标壳落地适配

一句话：
**03 负责把壳内前层经历池转成可继承资产。**

---

## 三、三者关系

### 1. 主权关系
- 01 不定义长期真相
- 02 定义长期真相
- 03 不篡权，只负责迁移工艺

### 2. 接壳顺序
顺序必须是：

1. 先定 01
2. 再定 02
3. 再定 03
4. 再接目标壳
5. 最后做前层校正

### 3. 入脑总流程
壳内前层经历池  
→ `08-IMPORT-STAGING`  
→ 分流  
→ 升格  
→ 进入 02 正式分层

---

## 四、母本与版本映射的边界

### 母本层负责什么
母本层只负责：

- 通用规则
- 通用分层
- 通用主权边界
- 通用接壳原则
- 通用迁移工艺

### 版本映射层负责什么
版本映射层才负责：

- 某个具体壳怎么接
- 某次实验壳的特殊差异
- 某一代壳的结构偏差
- 特定版本的首配差异
- 特定版本的落地步骤

### 硬口径
**母本总卡尽量不写具体版本号。**  
**具体版本差异，单独写入 03 的版本映射卡。**

---

## 五、当前施工同步口径

当前架构建设期间：

- `02`：暂停同步
- `01`：维持手动同步
- `03`：维持手动同步

本次收口后：

- `01`、`03` 各执行一次手动同步
- `02` 是否恢复同步，必须单独评估

---

## 六、当前施工目标

当前目标不是搬某一版壳内 memory。  
当前目标是：

1. 把 OpenClaw 原生记忆体系降为前层经历池
2. 让 01、02、03 形成稳定母本分工
3. 让 02 成为完整母脑真源
4. 让 03 成为可跨版本复用的迁移工艺层
5. 让以后换壳时主要改映射，不改主权

---

## 七、一句话总定义

本工程不是传承某版壳的 memory。  
而是以 01、02、03 为母本体系，  
把壳内原生记忆降为前层经历池，  
再通过 03 映射进入 02 的长期主权结构。
