# 03-首配阶段双模型多模型设置卡

## 定位
本卡用于定义：

**如果目标系统是双模型 / 多模型结构，必须在首配阶段正确完成设置。**

这不是后补优化项，
而是壳适配工艺的一部分。

## 当前已验证现实
在新壳实验中，已出现以下风险：

- 若首配阶段未正确完成双模型 / 多模型设置
- 系统会先落到默认单模型现实
- 或先落入默认 `anthropic` 模型族
- 后续再补第二模型时，容易继续在错误现场绕圈
- 表面上像“快成功了”，实际底层结构已经错位

因此：

**双模型 / 多模型的建立，必须前置到首配阶段。**

## 正式规则

### 规则 1
如果目标系统要求双模型 / 多模型，
必须在首配阶段完成结构性设置。

### 规则 2
不允许先按默认单模型走通，
再在后层临时补第二模型。

### 规则 3
首配阶段必须明确：
- 是否启用第二模型 / 第三模型
- 每个模型槽对应的目标模型族
- 不允许让系统自行落入默认模型族后再补救

### 规则 4
如果首配阶段未正确完成双模型 / 多模型结构，
则本轮不视为正式通过。

## 当前结论
双模型 / 多模型不是“后续增强项”，
而是：

**首配阶段的结构性工艺。**

## 一句话收口
如果目标是双模型 / 多模型，
就必须在首配阶段一次性做对；
否则后面大概率只是：
**在错误底座上继续补。**
