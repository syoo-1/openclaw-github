# 母层调用原则

## 目的
本文件用于规定：母层在什么情况下需要进入，什么情况下不需要进入。

目标是让母层保持为上位规矩，
而不是让每次 skill 调用都重复走一遍母层。

---

## 固定原则
母层用于定结构、定方法、定升格边界。

母层不是每次日常执行都要重新阅读的操作手册。

---

## 必须进入母层的情况
以下情况应进入母层：

1. 新建一个 skill
2. 重构一个 skill 的层级结构
3. 判断某项内容该进哪一层
4. 判断某项内容是否具备母层价值
5. 判断某项内容是否可以从实例层回写母层
6. 发现现有实例层结构已明显偏离母层方法

---

## 一般不必进入母层的情况
以下情况通常不必重新进入母层：

1. 日常调用某个已建立好的 skill
2. 按既有 runbook 执行当前任务
3. 补充某个实例层的 facts
4. 新增某个实例层的 correction
5. 在既有实例层中继续写 proposal
6. 查看某个实例层的总览与当前状态

---

## 日常调用的默认入口
一个 skill 日常调用时，默认入口应为：

1. 该 skill 的总览
2. 该 skill 的 `SKILL.md`
3. `50-HANDOFF-RUNBOOK/CURRENT-TASK.md`
4. `50-HANDOFF-RUNBOOK/EXECUTION-ORDER.md`

只有在出现结构争议、层级争议、升格争议时，才回到母层。

---

## 调用顺序原则
### 新建 skill 时
先看母层，再建实例层。

### 日常执行时
先看实例层总览，再进 handoff 执行入口。

### 出现结构争议时
回母层判断，再返回实例层继续。

---

## 一句话
母层是上位规矩，不是日常负担；  
定结构时看母层，日常执行按实例层入口走。
