# ENTRY-RULES

## 目的
本文件规定：什么内容适合进入 proposal 层，什么内容还不够格。

---

## 适合进入 proposal 层的情况
以下情况通常适合入 proposal：

1. 某个纠错已重复出现两次以上
2. 某项结论已超出单次实验范围
3. 某项变化会影响安装顺序、首配逻辑或验收口径
4. 某项经验已值得未来执行者复用
5. 某项问题不能只留在 correction 层，否则还会重复蒸发

---

## 暂不适合进入 proposal 层的情况
以下情况通常还不该入 proposal：

1. 只是一次性观察
2. 还没有形成修正结论
3. 还无法说明影响范围
4. 只是情绪化抱怨
5. 只是模糊感觉“好像该改”

---

## 最低纪律
提案不是升级捷径。
没有依据、没有影响范围、没有风险说明的内容，不得进入 proposal 层。

---

## 一句话
不是每个纠错都要提案，但凡值得制度化的纠错，都不能只停留在 correction 层。
