# 项目运行记录

## 1. 项目名称
双模型共脑与外部主脑系统

## 2. 当前阶段
基础架构已落地，进入“开始运转”阶段。

## 3. 本次更新摘要
已建立独立于 OpenClaw 的外部主脑、备份与恢复机制，以及 handoff / projects / longterm 三层模板，并形成第一份正式交接单。

## 4. 新增决定
- 采用 ~/Brain-Canonical 作为唯一主记忆仓库
- 采用 ~/Brain-Backups 保存主脑备份包
- 采用 ~/OpenClaw-Backups 保存 OpenClaw 快照包
- 采用 ~/backup-openclaw-brain.sh 作为一键备份脚本
- 采用 ~/restore-openclaw-brain.sh 作为一键恢复脚本
- 采用 handoff / projects / longterm 三层结构管理记忆
- 采用 archive 机制控制长期堆积
- 采用 open上 / kimi上 作为模型接班口令
- 采用 1~6 作为固定模式码

## 5. 变更记录
- 不再把 sessions/*.jsonl 视为唯一主记忆
- 不再把 OpenClaw 本体视为主脑本体
- 外部备份不再只是副本，而是提升为主脑真源

## 6. 已验证项
- Brain-Canonical 目录已建立
- Brain-Canonical 备份包已生成并验证可读取
- OpenClaw 快照包已生成
- 一键备份脚本已跑通
- 一键恢复脚本已创建
- handoff 模板已建立
- projects 模板已建立
- longterm 模板已建立
- 第一份正式 handoff 已建立

## 7. 待验证项
- 恢复脚本的完整实战恢复演练
- 双模型自动切换与自动交接
- 手动切换提示模式的实际接入
- longterm 条目的实际准入流程
- 异机备份方案

## 8. 阻塞点
- 当前备份仍在同一台 Mac mini 上
- 自动化机制尚未接入实际运行
- 三层结构虽已建立，但正式内容仍偏少

## 9. 后续计划
- 建立第一条正式长期记忆
- 把切换规则整理成独立规则页
- 逐步把真实项目内容写入 handoff / projects / longterm
- 视情况再进入自动化阶段
- 后续补上异机备份

## 10. 候选长期记忆
- Brain-Canonical 是唯一主记忆本体
- OpenClaw 只是工作镜像与运行层
- sessions/*.jsonl 不应作为唯一主记忆真源
- 切换模型时应先交接，再切换，再继续
