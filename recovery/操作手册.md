# 操作手册

## 一、先记住这几个核心位置
- 主脑仓库：~/Brain-Canonical
- 主脑备份：~/claw/current/backups/Brain-Backups
- OpenClaw 快照：~/claw/current/backups/OpenClaw-Backups
- 恢复说明：~/Brain-Canonical/recovery/RESTORE.md

## 二、日常最常用的命令
### 1. 本机快照备份
~/claw/current/scripts/backup-openclaw-brain.sh

### 2. 主脑推送到 GitHub
~/claw/current/scripts/push-brain-canonical.sh "更新说明"

### 3. 恢复时看的说明
打开：
~/Brain-Canonical/recovery/RESTORE.md

## 三、出问题时先判断
### 情况 A：OpenClaw 出问题，但本机还在
先不要乱重装。
先确认：
- ~/Brain-Canonical 还在不在
- ~/claw/current/backups/Brain-Backups 里最近的包在不在
- ~/claw/current/backups/OpenClaw-Backups 里最近的包在不在

### 情况 B：主脑还在，但 OpenClaw 乱了
优先保住 Brain-Canonical，不要先翻 sessions。
先看：
- ~/Brain-Canonical/recovery/RESTORE.md

### 情况 C：本机坏了，但 GitHub 还在
先从 GitHub 恢复 Brain-Canonical。
再考虑重建 OpenClaw 外壳。

## 四、恢复顺序
1. 先确认 Brain-Canonical
2. 再确认 Brain-Backups
3. 再确认 OpenClaw-Backups
4. 最后才处理 ~/.openclaw 的运行层

## 五、禁止事项
- 不要先删备份再验证
- 不要把 sessions/*.jsonl 当作唯一主记忆
- 不要一出问题就马上重装
- 不要把未经确认的内容直接写入长期记忆

## 七、当前固定规则
- Brain-Canonical 是唯一主记忆本体
- OpenClaw 是工作镜像和运行层
- 切换模型前先交接
- 有重要更新后再 push 主脑
