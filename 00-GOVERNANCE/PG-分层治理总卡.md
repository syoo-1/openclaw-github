# PG-分层治理总卡

## 目的
把 PG 从“已有规则的监管脑”升级成“有分层治理能力的监管脑”。

## 当前原则
1. PG 保持独立脑，不并回 Core
2. PG 吸收 02 的分层治理思想，不照搬 Core 旧遗产
3. 正式规则进内围，旧材料降格为参考
4. 临时层、导入层、退役层分开
5. PG 继续服务监管主线，不变成普通工作脑

## 当前分层
- 00-GOVERNANCE：治理规则、分层原则、准入规则
- 02-MAINLINE：当前监管主线、当前阶段、当前停点
- 03-LONGTERM：长期稳定监管规则、长期有效判断
- 07-RETIRING：待退役但暂不删除的旧规则/旧材料
- 08-IMPORT-STAGING：待筛选导入材料
- 99-DEFERRED：延后处理材料

## 禁止
- 不把旧 handoff / trigger 直接当正式规则
- 不把运行现场短材料直接升格为长期层
- 不把 Core 专属材料当成 PG 正式主权来源
