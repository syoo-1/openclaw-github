# PG 自动触发当前现实

## 当前已确认
PG 当前不是没有自动触发链。

已经存在一条最小守望链，用于检查 Core 的核心端口是否仍在监听：
- 当前检查端口：`38789`
- 当前脚本：`~/SYOO1-PG/runtime/pg-guard-core-v1.sh`

## 当前能力边界
这条链当前只能做到：

1. 检查 Core 端口是否监听
2. 端口异常时等待一次
3. 仍异常时执行一次 kickstart
4. 再次检查端口是否恢复

## 当前不能做到
这条链当前还**不能**做到：

1. 判断 Core 前台是否语无伦次
2. 判断 Core 是否发生逻辑失常
3. 判断 Core 是否“端口还活着，但回答链已漂”
4. 判断 Core 是否“无异常日志，但实际已失常”
5. 对前台回答质量异常做自动触发

## 当前结论
PG 当前已经具备：
**最小端口级守望能力**

但还**不具备**：
**逻辑失常级守望能力**

因此：

**“PG 已有自动触发链”**
这句话如果不加限定，容易说过头。

更准确的说法应是：

**PG 当前只有端口级守望，不等于已经具备完整自动触发能力。**

## 一句话收口
PG 当前能守“端口死没死”，还不能守“前台逻辑有没有失常”。
