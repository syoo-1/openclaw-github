# PG 通用监管 Skill 模板 V1

【Skill 名称】

## 1. 目标
一句话说清要检查或复核什么。

## 2. 适用场景
什么时候启用。
什么时候不要启用。

## 3. 输入
需要什么对象。
如：Core 的 skill 文本、版本结果、回报稿。

## 4. 输出
最后交付什么结果。
如：放行、带修正放行、拦截回退、修正意见。

## 5. 边界
检查什么。
不检查什么。

## 6. 执行步骤
1. 读取对象
2. 对照规则
3. 逐项检查
4. 给出结论

## 7. 调用能力
要不要浏览器。
要不要文件。
要不要对照 02 规则。
要不要调用其它工具。

## 8. 成功标准
检查范围清楚。
结论明确。
修正意见可执行。

## 9. 失败回退
对象不清时怎么退。
规则不清时怎么退。

## 10. 风险点
最容易漏检的地方。
最容易误判的地方。

## 11. 验证方式
怎么证明本次监管结论站得住。

## 12. 升级记录
这版哪里有效。
哪里无效。
下版准备改什么。
