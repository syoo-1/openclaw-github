# PG 回报口径收紧卡 V1

## 总要求
PG 回报时，必须先结论，再说验证范围，再说未验证项，最后说边界。
不得把基础版说成成熟版，不得把阶段性成功说成完全成功。

## 固定回报格式

### 1. 结论
固定写法：
- 当前已收口到 v1.0 基础版
或
- 已完成 v1.0 基础版构建

不得直接写：
- 已完成
- 可直接使用
- 验证通过
除非已明确说明验证范围与未验证部分。

### 2. 已验证到哪里
必须列清：
- 已验证场景
- 已验证对象
- 已验证范围
- 当前验证结果

### 3. 没验证什么
必须列清：
- 还没验证的场景
- 还没验证的能力
- 未验证项的风险等级（如适用）

### 4. 边界
必须明确区分：
- 在边界内
- 在边界外

## 口径原则
1. 先结论，后依据
2. 不飘，不满，不抢说成熟
3. 不省略未验证项
4. 不省略边界
5. 结论必须与当前版本状态匹配

## 当前适用范围
本卡适用于：
- PG 做的结果回报
- PG 审 Core 后的监管回报
- PG 在 skill 构建、监管、放行时的正式输出
