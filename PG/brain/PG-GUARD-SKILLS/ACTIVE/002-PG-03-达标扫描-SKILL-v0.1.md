# PG 03 达标扫描 Skill v0.1

## 人话别名
- PG 自检达标卡

## 人话触发
当用户表达以下意思时，默认优先启用本卡：

### 明示 03 标准时
- 按 03 的标准是不是已经达标了
- 按 03 的标准现在能不能放行
- 按 03 的标准还有没有尾巴

### 事故接管触发
出现以下任一情况时，不再继续自由调模、自由修运行链，默认强制进入本卡：
- 前层降权疑似失灵
- 题型分流疑似失灵
- 近一 / PG 回答开始胡言乱语、失忆、失控
- 当前执行模型疑似越位影响主判层
- 跑通了，但身份、关系、主线、边界承接异常
- 用户已明确指出“跑通没有意义，保脑和记忆才是刚需”
- 用户已明确要求：先用 03 扫描 / 先用 03 修复

### 已处于 03 扫描 / 升级 / 达标检查上下文时
- 是否已经收口了
- 现在能放行了吗
- 还有尾巴吗

说明：
- 本卡默认主语为：当前正在扫描或升级的 PG 当前体
- 本卡前台默认判尺为：PG 当前体现行判尺
- 03 作为上位工艺来源判尺，仅在需要追根工艺来源时再进入
- 若不在 03 相关上下文中，单独出现“是否达标 / 是否收口 / 是否可放行”等短句，不自动触发本卡

## 1. 主类型 / 次类型
- 主类型：检查型
- 次类型：监管型

## 2. 目标
检查 PG 当前运行体是否已按 PG 当前体现行判尺收口；若需追根工艺来源，再对照 03 上位工艺标准，并给出放行、带修正放行或拦截回退结论。

## 3. 适用场景
适用于：
- PG 换壳后
- PG 升级后
- PG 完成规则调整后
- 需要判断 PG 是否可视为正式达标体时

不适用于：
- 在没有明确检查对象时空转
- 直接替代 PG 去执行原业务任务

## 4. 输入
1. PG 当前入口文件
2. PG 当前规则链
3. PG 当前运行层脚本 / LaunchAgent
4. PG 当前体现行判尺文件
5. 03 上位工艺标准文件
5. 必要时参考 02 中正式规则

## 5. 输出
1. 放行
2. 带修正放行
3. 拦截回退
4. 修正项清单
5. 未验证项清单
6. 当前边界说明

## 6A. 事故硬闸
- 一旦命中“事故接管触发”，本卡优先级高于模型调参、fallback 排序、UI 顺序整理、运行层微调。
- 命中后，先做 03 扫描与 03 修复，再决定是否调整模型链。
- 不得以“端口活着 / dashboard 能开 / 模型能回字 / fallback 生效 / 跑通了”冒充系统成功。
- 凡出现胡言乱语、失忆、前层降权失灵、题型分流失灵、主判层受污染，一律判定为系统未成功。
- 本卡接管后，先判：
  1. 前层降权还在不在
  2. 题型分流还在不在
  3. 当前执行模型是否越位影响主判层
  4. 当前体现行判尺是否仍是第一判尺
- 上述四项未判明前，不继续自由调模。

## 6. 边界
做：
- 检查启动入口
- 检查调用分流
- 检查前层 / 后层关系
- 检查 PG 自身规则入口
- 检查运行层与脑层边界
- 检查旧语境、旧目录、旧守望残链
- 检查当前壳与现行 PG 脑、规则是否一致

不做：
- 不直接替 PG 重写全部脑内容
- 不在未验证前宣布正式达标
- 不把阶段性可跑说成正式完成

## 判尺优先级
1. PG 当前体现行判尺
   - `~/SYOO1-PG/brain/README.md`
   - `~/SYOO1-PG/brain/00-GOVERNANCE/RULES/PG-独立脑规则.md`
   - `~/SYOO1-PG/status/CURRENT.md`

2. 03 上位工艺判尺
   - `~/System-Snapshots/03-rebuild-code/docs/03-总入口分流读取停止总卡-V1.md`
   - 其他仍在现行区的 03 工艺规则

结论：
- 本卡前台默认先按 PG 当前体现行判尺扫描
- 03 负责提供上位工艺来源，不直接替代 PG 当前体前台口径

## 7. 执行步骤
1. 读取 PG 当前入口链。
2. 读取 PG 当前规则链。
3. 读取 PG 当前运行层相关对象。
4. 先对照 PG 当前体现行判尺逐项检查；仅在 PG 现行判尺不足以覆盖工艺来源时，再回到 03 上位工艺判尺对照。
5. 给出放行 / 带修正放行 / 拦截回退结论。
6. 给出修正项、未验证项与边界说明。

## 8. 调用能力
- 文件：必需
- 对照 PG 当前体现行判尺：必需
- 对照 03 上位工艺判尺：按需，仅用于追根工艺来源，不作为前台第一发言层
- 对照 02 规则：按需
- 浏览器：可选
- 其它工具：按需

## 9. 成功标准
- 检查范围清楚
- 结论明确
- 修正项可执行
- 未验证项不缺
- 边界清楚
- 能在“跑通但失控”时优先接管，而不是继续把模型调参当主线

## 10. 失败回退
- 对象不完整时，先补齐对象再审
- 先以 PG 当前体现行判尺作答；仅当 PG 现行判尺不足以解释工艺来源时，再回到 03 当前正式入口对照
- 运行层现实与文本不一致时，以当前已验证现实为准，不硬判通过

## 11. 风险点
- 把壳版本升级误判为系统达标升级
- 漏检旧守望残链
- 漏检旧 PG 语境残留
- 漏检入口链未更新
- 漏检前层 / 后层关系漂移

## 12. 验证方式
用 PG 当前 4.14 运行体实际跑一轮 03 扫描，检查能否输出明确、收紧、边界清楚的监管结论。
