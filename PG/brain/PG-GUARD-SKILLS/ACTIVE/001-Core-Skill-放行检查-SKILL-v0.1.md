# Core Skill 放行检查 Skill v0.1

## 1. 主类型 / 次类型
- 主类型：检查型
- 次类型：执行型

## 2. 目标
检查 Core 新构建或更新的 skill 是否符合 02 已立住的规则，并给出放行结论。

## 3. 适用场景
适用于：
- Core 新建一个 skill 后
- Core 更新一个 skill 后
- 需要判断是否允许继续使用时

不适用于：
- 直接替代 Core 去做原任务
- 在没有对象可检查时空转

## 4. 输入
1. Core 的 skill 回报稿
2. Core 的版本结果
3. 02 中相关规则

## 5. 输出
1. 放行
2. 带修正放行
3. 拦截回退
4. 具体修正意见

## 6. 边界
做：
- 检查类型
- 检查版本口径
- 检查边界
- 检查未验证部分
- 检查风险点

不做：
- 不直接替 Core 重写全部 skill
- 不把监管结果混装进 02 正文

## 7. 执行步骤
1. 读取 Core skill 回报稿。
2. 对照 02 规则。
3. 逐项检查关键点。
4. 给出放行结论。
5. 给出必要修正意见。

## 8. 调用能力
- 文件：必需
- 对照 02 规则：必需
- 浏览器：可选
- 其它工具：按需

## 9. 成功标准
- 结论清楚
- 修正点清楚
- 能明确判断是否放行

## 10. 失败回退
- 对象不完整时，要求补齐后再审
- 规则不明确时，先回到 02 对照

## 11. 风险点
- 漏检类型错误
- 漏检版本口径过满
- 漏检未验证部分缺失

## 12. 验证方式
用已完成的 Core skill 回报稿跑一轮，检查能否得出明确结论。
