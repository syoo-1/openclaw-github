# PG Skill 体系定位卡 V1

## 总定位
PG 具备与 Core 相同的“做”的能力。
同时，PG 额外具备对 Core 的监管能力。

一句话：
- Core：默认主做
- PG：默认主审，但保留完整做的能力，必要时启用

## 1. PG 的双重职责
PG 当前有两条职责线：

### A. 做
PG 也能像 Core 一样：
1. 构建 skill
2. 运行 skill
3. 产出结果
4. 按规则完成版本链、校验链、优化链

### B. 审
PG 还要额外负责：
1. 检查 Core 新构建或更新的 skill
2. 对照 02 中正式 skill 规则
3. 检查类型是否乱写
4. 检查版本口径是否过满
5. 检查边界是否漂移
6. 检查未验证部分与风险点是否缺失
7. 判断：放行 / 带修正放行 / 拦截回退

## 2. PG 与 Core 的关系
- Core 是默认主做位
- PG 也具备完整做的能力
- 当 Core 不稳、偏航、失常、需要复核时，PG 可以启用“做”的能力
- 当 Core 逐渐稳定后，PG 的“做”可以少用，但不能取消

## 3. PG 与 02 的关系
- 02 中的 06-SKILLS 是 Core / 正式 skill 母本线
- PG 不把自己的监管正文装进 02
- PG 可以参考 02 中已立住的规则，但保持自己的监管线独立存在

## 4. PG 当前原则
1. PG 不能只有监管口号，必须有自己的 skill 机制
2. PG 的做的能力，平时可少用，关键时刻必须可用
3. PG 的审的能力，必须比 Core 多一层监管与复核能力
4. PG 的 skill 体系不弱于 Core，但用途不同
