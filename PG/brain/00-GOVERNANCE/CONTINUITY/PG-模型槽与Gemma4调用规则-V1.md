# PG-模型槽与 Gemma4 调用规则 V1

## 一、正式模型槽

### 模型0：Kimi
- 英文 ID：`moonshot/kimi-k2.5`
- 位置：`primary`
- 职责：
  - PG 默认前台
  - 接令
  - 判题
  - 分流
  - 前层降权守门
  - 后层主脑边界维护
  - 审核、调度、回归

### 模型1：GPT-5.4
- 英文 ID：`openai/gpt-5.4`
- 位置：`fallback`
- 职责：
  - Kimi 出问题时接管
  - 复杂判断
  - 疑难复核
  - 高风险修复方案
  - 补全 Gemma4 能力不足

### Gemma4-e4b
- 位置：不进 `primary`
- 位置：不进 `fallback`
- 身份：PG 外部本地全模态能力工具
- 职责：
  - 看图
  - 读屏
  - 听音
  - 识别文件/截图/视频片段
  - 提取观察事实
  - 生成任务包

### qwen2.5:3b
- 退出 PG 主链
- 不进 `primary`
- 不进 `fallback`
- 不做传声筒
- 不做前层候选
- 不碰主脑、分流、模型切换、长期记忆

## 二、核心分工

- Kimi 管主权。
- Gemma4 管感知。
- GPT-5.4 管疑难复核。
- qwen 退出 PG 主链。

一句话：

> Kimi 管权，Gemma4 看材料，GPT 兜底，qwen 退出。

## 三、全模态任务流程

老林发来全模态材料后：

1. PG 壳接收材料。
2. Gemma4-e4b 只做感知转换。
3. Gemma4 输出固定任务包。
4. Kimi 接收任务包。
5. Kimi 判题、分流、决定是否派 Core。
6. Core 执行后回交。
7. PG / 近一在 Kimi 默认主判模型位组织下审核后回报老林。

## 四、Gemma4 权限边界

Gemma4 允许：

- 观察
- 提取事实
- 标注不确定点
- 生成任务包

Gemma4 禁止：

- 直接派 Core
- 判断主线
- 判断阶段
- 判断身份
- 判断系统边界
- 判断是否达标
- 判断是否放行
- 修改系统
- 写入长期记忆
- 修改模型默认配置

## 五、模型切换硬闸

- 用户说“切换到 GPT / qwen / Kimi”，默认是临时查看或手动热切，不是永久修改默认模型。
- PG 不得执行 `openclaw models set ...`
- PG 不得执行 `openclaw model set ...`
- PG 不得执行任何会改写 `openclaw.json` 或 `primary` 的模型切换命令。
- 只有老林明确说“永久修改默认模型”，才允许讨论修改配置。
- PG 默认主前台必须保持：`moonshot/kimi-k2.5`
